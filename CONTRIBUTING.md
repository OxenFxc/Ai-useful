# Contributing to Super Source Toolbox

Thank you for your interest in contributing to Super Source Toolbox! We welcome contributions from the community to make this tool better.

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/your-username/super-source.git
    ```
3.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```
4.  **Make your changes**.
5.  **Commit your changes** with descriptive commit messages.
6.  **Push your branch** to your fork:
    ```bash
    git push origin feature/my-new-feature
    ```
7.  **Open a Pull Request** on the original repository.

## Coding Guidelines

*   **Language**: The project is written in C.
*   **Style**: Please follow the existing coding style (indentation, variable naming, etc.).
*   **Modularity**: Keep code modular. New features should ideally be implemented as separate modules in `src/modules/` or tools in `src/toolbox/`.
*   **Comments**: Add comments to explain complex logic.
*   **Testing**: Test your changes locally to ensure they work as expected. Verify that the project builds successfully with `make`.

## Reporting Bugs

If you find a bug, please open an issue on GitHub. Include:
*   Steps to reproduce the bug.
*   Expected behavior.
*   Actual behavior.
*   Your system information (OS version, etc.).

## Feature Requests

We welcome feature requests! Please open an issue to discuss your idea before implementing it.
